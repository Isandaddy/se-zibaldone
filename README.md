# 🏛️ Zibaldone: Engineering Archives

[🌐 日本語版 (README_jp.md)](./README_jp.md)

매번 일본 전역의 다양한 현장(現場)을 이동하며 프로젝트를 수행하는 프리랜서 SE(시스템 엔지니어)로서, 그동안 치열하게 겪고 극복해 온 실전 노하우와 장인정신을 기록하는 나만의 '지식 금고'이자 외장 지식 저장소입니다.

르네상스 천재들의 잡록인 **'치발도네(Zibaldone)'**처럼, 파편화되어 사라지기 쉬운 현장의 생생한 관찰과 보편적 규칙들을 정교하게 구조화하여 축적합니다.

---

## 🎯 저장소의 목적 (Objective)

1. **지식의 영속성 확보 및 복잡도 하향**
   - 현장이 바뀔 때마다 소멸하기 쉬운 소중한 기술 자산과 해결 프로세스를 기록합니다. 다음 프로젝트에서도 시행착오 없이 즉시 이식하여 사용할 수 있는 보편적인 베스트 프랙티스를 구축합니다.

2. **철저한 익명성과 범용성**
   - 보안과 개인정보 보호를 최우선으로 합니다. 특정 회사명, 프로젝트명, 개인정보 및 보안 민감 정보는 철저히 배제하고, 어떠한 인프라 환경에서도 재사용 가능한 순수 기술 패턴과 구조 중심의 보편적 내용만을 엄선하여 문서화합니다.

3. **생성형 AI 주도 개발의 표준 나침반 (AI-Driven Reference)**
   - 본 저장소의 규칙들은 특정 AI 도구에 종속되지 않는 범용적인 자연어 지침입니다. Amazon Q, Gemini, ChatGPT, Claude 등 그 어떤 생성형 AI 모델에 프롬프트로 주입하더라도 완벽하게 연동되어, 나를 위한 최고급 퀄리티의 소스코드를 단 한 번에 생산하도록 통제하는 강력한 안전핀 역할을 수행합니다.

## 📂 저장소 및 자산 구조 (Directory Structure)

본 저장소는 현장 이동 시 가장 먼저 확인하고 적용할 수 있도록 실용적으로 분류하여 독립된 파일 단위로 관리합니다.

### 📁 field-insights/

- **`📄 feedback-notebook.md`** : 일본 엔지니어링 현장에서 지적받거나 스스로 피드백한 실수 방지용 기록장. 커뮤니케이션, 보고 방식, 공지 엄수 등 프로 프리랜서의 신뢰를 지키기 위한 실시간 액션 아이템 리스트.

### 📁 ai-reusable-assets/ (AI 주입용 5대 표준 강령)

- **`📄 01-security-and-conflict.md`** : 자격 증명 하드코딩 금지, 로그 내 개인정보 마스킹, HttpOnly 기반 Secure 인증 토큰 관리 등 인프라 보안 금기 사항 및 룰 충돌 시 행동 지침.
- **`📄 02-backend-architecture.md`** : Express 4계층(`Route-Controller-Service-Repository`) 단방향 의존성 구조 제어, 입력 검증 및 에러 유출 차단, N+1 루프 쿼리 금지, 코드 중첩 Depth 2 제한, 설명 변수 분해 및 JSDoc 문서화 표준.
- **`📄 03-frontend-architecture.md`** : Next.js App Router 환경의 `"use client"` 최소화 및 파일 단위 물리 격리, 컴포넌트 내 fetch 직접 작성 금지, UI/UX 데이터 3대 상태(Loading/Error/Empty) 구현, 이중 송신 방지, 폼 초기화 및 컨펌 다이얼로그 가드, URL 쿼리 파라미터 동기화, `clsx/cn` 통일, 브라우저 환경변수 보안.
- **`📄 04-testing-guidelines.md`** : 테스트 케이스 체언종결 및 추상적 표현 금지, `Given-When-Then` 구조 시각화, 기능 테스트 내 완전한 목업(Mock) 환경 격리, 화면 테스트용 고유 셀렉터(`data-testid`) 강제, 리액트 비트리거 회피 및 유니크 제약 필드의 동적 데이터 생성 규칙.
- **`📄 05-review-checklist.md`** : AI 코드 생성 프로세스 통제(착수 전 확인 3단계), 기존 파일 수정 시 영향 범위 조사 및 주석-코드 동시 업데이트 규칙, 도메인 업무와 기술적 판단의 경계 준수, 규격화된 TODO 마킹, Git Rebase 충돌 해결 및 자동 생성 파일(DB 스키마, 패키지 락) 재생성 원칙.
