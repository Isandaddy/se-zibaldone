# 🛑 보안 관련 금기사항 및 공통 룰 충돌 시 액션

> 어떤 프로젝트나 현장으로 이동하더라도 보안을 준수하면서 AI의 생산성을 극대화하기 위한 최우선 보안 규칙입니다.

## ⚖️ 1. 공통 룰 충돌 시 액션 (Conflict Resolution)

- **인프라 프로퍼 최우선**: 현장의 보안 규정 및 인프라 프로퍼(Proper)의 지침이 본 문서의 AI 가이드라인보다 항상 최우선합니다.
- **자의적 판단 금지**: 사내 규정과 본 문서의 룰이 충돌하거나 애매한 영역이 발생하면, 즉시 작업을 중단하고 담당 프로퍼에게 확인을 요청합니다.

## 🔒 2. 자격 증명 및 비밀 정보 하드코딩 전면 금지 (Secret Management)

- **적용 가이드**: API 키, 데이터베이스(PostgreSQL) 접속 문자열, 패스워드, 암호화 키 등 모든 민감한 자격 증명(Credentials)을 소스코드 내에 직접 기재(Hardcoding)하는 것을 전면 금지합니다.
- **실장 방식**:
  - 모든 비밀 정보는 반드시 환경 변수(`process.env`)를 통해 주입받도록 코드를 작성합니다.
  - 로컬 개발 및 DevContainer 환경에서는 `.env` 또는 `.env.local` 파일을 활용하며, 이 파일들이 Git 저장소에 커밋되지 않도록 `.gitignore` 설정을 철저히 유지합니다.
  - 스테이징(AWS) 환경의 실제 환경 변수 및 자격 증명 관리는 인프라 담당 프로퍼(Proper)의 가이드와 인프라 아키텍처 규칙을 따릅니다.

### 📌 규칙 03. 로그 내 개인정보 및 토큰 출력 금지 및 마스킹 처리 (Secure Logging)

- **적용 가이드**: 애플리케이션 로그(콘솔 로그, 파일 로그 등)에 사용자의 민감한 개인정보나 시스템 인증에 사용되는 토큰/비밀번호 정보가 평문으로 기록되는 것을 전면 금지합니다.
- **보안 금지 사항**: 로그 출력 시 아래 데이터가 그대로 노출되지 않도록 차단합니다.
  - 개인정보 (이름, 이메일, 전화번호, 생년월일 등)
  - 인증 정보 (Access/Refresh Token, 세션 ID, 비밀번호, API Key 등)
- **실장 방식**:
  - 로그를 출력하기 전, 해당 객체나 문자열에서 민감한 정보를 필터링하는 전역 마스킹(Masking) 유틸리티나 미들웨어를 거치도록 합니다.
  - 마스킹은 알아볼 수 없는 형태(예: `tes***@example.com`, `eyJhbGciOi...[MASKED]`)로 치환하여 기록합니다.
  - 개발용 환경(DevContainer)이라 할지라도 실운영 환경(AWS)과 동일한 마스킹 규칙을 적용하여 관성적인 실수를 예방합니다.

### 📌 규칙 04. 인증 토큰의 HttpOnly 쿠키 관리 의무화 (Secure Token Management)

- **적용 가이드**: 사용자 로그인 및 인증 체계에 사용되는 핵심 토큰(Access Token, Refresh Token 등)을 브라우저의 로컬 스토리지(LocalStorage), 세션 스토리지(SessionStorage) 또는 자바스크립트로 접근 가능한 일반 쿠키에 저장하는 것을 전면 금지합니다.
- **보안 목적**: 자바스크립트 코드가 인증 토큰에 직접 접근하는 것을 원천 차단하여, 악성 스크립트 주입 공격(XSS)이 발생하더라도 해커가 토큰을 탈취하여 대포 계정으로 도용하는 사고를 물림적으로 방지하기 위함입니다.
- **실장 방식 (프론트·백엔드 공통 가이드)**:
  - **백엔드(Express)**: 로그인 및 토큰 갱신 API 성공 시, 토큰을 HTTP 응답 바디(JSON)로 내려주지 말고 반드시 HTTP 응답 헤더의 `Set-Cookie`를 통해 발급합니다. 이때 쿠키 옵션에 **`httpOnly: true`**, **`secure: true`**(AWS 스테이징/운영 환경 필수), **`sameSite: 'lax'` 또는 `'strict'`** 설정을 강제합니다.
  - **프론트엔드(Next.js)**: API 호출 시 자바스크립트로 토큰을 꺼내 헤더에 수동으로 실어 보내지 않고, 브라우저가 네트워크 요청 시 쿠키를 자동으로 백엔드에 전송하도록 `fetch`나 `axios` 옵션에 `credentials: 'include'`(또는 `withCredentials: true`) 설정을 집약 관리(lib/api 규칙 준수)합니다.
