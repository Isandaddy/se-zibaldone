# 🏛️ Zibaldone: Engineering Archives (日本語版)

[🌐 Korean Version (README.md)](./README.md)

日本全国の様々な現場を渡り歩きながらプロジェクトを遂行するフリーランスSE（システムエンジニア）として, これまで現場で培ってきた実践的なノウハウや職人精神を記録する, 自分だけの「ナレッジベース（知識の金庫）」であり外部記憶ストレージです。

ルネサンス期の天才たちが現場での観察や知識を書き留めていた雑記帳 **「ジバルドーネ（Zibaldone）」** のように, 現場ごとに断片化して失われがちな生々しい観察結果と普遍的な開発ルールを精緻に構造化して蓄積します。

---

## 🎯 本リポジトリの目的 (Objective)

1. **ナレッジの永続性確保と複雑度の低減**
   - 現場が変わるたびに消滅しやすい貴重な技術資産やトラブルシューティングのプロセスを記録します。次のプロジェクトでも試行錯誤することなく、即座に移植して利用できる普遍的なベストプラクティスを構築します。

2. **徹底した匿名性と汎用性の追求**
   - セキュリティと個人情報保護を最優先とします。特定の会社名、プロジェクト名、個人情報、およびセキュリティ機密情報は徹底的に排除し、いかなるインフラ環境でも再利用可能な「純粋な技術パターン」と「構造」中心の普遍的な内容のみを厳選して文書化します。

3. **生成AI駆動開発（AI-Driven）の標準コンパス**
   - 本リポジトリの規則は、特定のAIツール（Amazon Q、Gemini、ChatGPT、Claude、Copilotなど）に依存しない普遍的な自然言語の指示書です。どの生成AIモデルにプロンプトとして投入しても完璧に連動し、自分向けに最高品質のソースコードを一発で生成するように統制する強力な安全弁（ガードレール）の役割を果たします。

## 📂 ディレクトリおよび資産構造 (Directory Structure)

本リポジトリは、新しい現場に参画した際に最も早く確認して適用できるよう、実用的に分類して独立したファイル単位で管理しています。

### 📁 field-insights/

- **`📄 feedback-notebook.md`** : 日本の現場で指摘された内容や、自らフィードバックしたミス防止用の記録帳。コミュニケーション、報告方法、アナウンス遵守など、プロフェッショナルとしての信頼を守るためのリアルタイム・アクションアイテムリスト。

### 📁 ai-reusable-assets/ (AI注入用の5大標準綱領)

- **`📄 01-security-and-conflict.md`** : 認証情報のハードコーディング禁止、ログ内の個人情報マスキング、HttpOnlyベースのSecure認証トークン管理など、インフラセキュリティの禁忌事項およびルール衝突時の行動指針。
- **`📄 02-backend-architecture.md`** : Express 4層（`Route-Controller-Service-Repository`）単方向依存関係の構造制御、入力バリデーションおよびエラー漏洩遮断、N+1ループクエリの禁止、コードネストDepth 2制限、説明変数の分解、およびJSDocドキュメント化標準。
- **`📄 03-frontend-architecture.md`** : Next.js App Router環境における `"use client"` の最小化およびファイル単位の物理隔離、コンポーネント内 fetch 直接記述の禁止、UI/UXデータ3大状態（Loading/Error/Empty）の実装、二重送信防止、フォーム初期化および確認ダイアログのガード、URLクエリパラメータの同期、`clsx/cn`の統一、ブラウザ環境変数のセキュリティ。
- **`📄 04-testing-guidelines.md`** : テストケースの体言止めおよび抽象的表現の禁止、`Given-When-Then`構造の視覚化、機能テストにおける完全なモック（Mock）環境隔離、画面テスト用の固有セレクタ（`data-testid`）の強制、Reactの非トリガー（Non-Trigger）現象回避、およびユニーク制約フィールドの動的データ生成規則。
- **`📄 05-review-checklist.md`** : AIコード生成プロセスの統制（着手前確認3段階）、既存ファイル修正時の影響範囲調査およびコメント・コード同時アップデート規則、ドメイン業務と技術的判断の境界遵守、規格化されたTODOマーキング、Git Rebaseによるコンフリクト解消、および自動生成ファイル（DBスキーマ、パッケージロック）の再生成原則。
